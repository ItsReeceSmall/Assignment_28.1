(P2) Explain the user side and server side factors that influence the performance of a website

Client Side

Download Speed

The most important factor to take into consideration when looking at a website is the speed at which it is downloaded to the clients machine. Nowadays fibre optic broadband is a norm so high download speeds are common. The simple thing to consider here is the slower the Internet connection; the longer it will take to download the web page. Comparing current fibre optic speeds to older dial up modem speeds, the time difference of loading the web page would be greatly significant. Dial up was only really capable at loading smaller web pages but still at a long duration.

Processor Speed

The speed of a processor is vital when it comes to processing web pages being processed by the browser. The slower the processor, the longer it will take the browser to process the incoming information and convert it to a web page, to then display on screen. Even though processor speed is not a big issue due to the advances in technology these days, this problem can still come up with older lower end systems; some can still be purchased today. In conclusion it is not really a factor to worry about, as a large majority of modern systems are fast. On the other hand, some browsers require large amounts of RAM to run along with a lot of the processors power too, this in effect can slow the PCs speed and performance, along with the performance of a website when it is being viewed.

Cache Memory

According to PCMag.com, Cache memory means; 'A temporary storage area in memory or on disk that holds the most recently downloaded Web pages. As you jump from Web page to Web page, caching those pages in memory lets you quickly go back to a page without having to download it from the Web again. In order to ensure that the latest page is displayed, the browser compares the dates of the cached page with the current Web page. If the Web page has not changed, the cached page is displayed immediately. If the Web page has changed, it is downloaded, displayed and cached.' (PCMag, 2015)

This type of memory is temporary memory used as storage which holds webpages in the clients system, browsers utilize this memory if you are constantly loading data from the same website o page. Obviously over time the cache will build up in size and will be updated when it needs to be through updates. So this process is a trade-off that client browsers have to make, they either load a page in shorter times or have a slow system, as the cache gets lager in size.

Browser

This factor is key also. Most browsers are automatically updated so they remain fast and responsive on nearly every machine, but there are occasions when they can have the difficulty of compatibility. Compatibility issues that can occur could be issues with interpreting and displaying of the page; browser is not powerful enough to show content in real time etc. Most commonly used browsers such as Google Chrome do not seem to have such problems but in browsers such as Internet Explorer these problems do arise as earlier versions of the browser were very slow and performance was affected negatively.

Server Side

Bandwidth Available

The available bandwidth of a site is of great importance as it states the maximum amount of traffic the site can receive at a given time; there is not anything the client user can really do about it. As en example, lets say the sites bandwidth is equivalent to a dial-up modem, around a few kilobytes a second, no matter how fast the users machines connection is, the download process of the page can only go as fast as the webserver sending the page, this important factor should be taken into consideration as it can highly affect website performance.

Executions to be performed before load

This factor effects the time it takes for the web page to respond to the user, providing that the users connection is faster than the web servers. In simpler terms, anything that has to be loaded before the web page can run, will cause delay for the client and make the load times of the site slower. This could possibly be PHP scripts being ran so the site is viewed properly on the clients end, even though this isn't that long of a process, all factors relating to the time taken to load must be considered. Looking further into this, server side scripts that take a long time will bottleneck the websites performance for both the webserver and the client viewer.

File Types e.g. GIF, WAV, SWF, AVI, Bitmap

The example file types mentioned in the heading above are a handful of the file types that are common culprits for performance issues on a website's web server. As an example say the user requests to load a web page which contains a small sized low resolution JPEG file, this page wow load relatively fast providing that the users connection is reliable and fast also. However, on the opposite end of the scale would be a web page containing an AVI video file, of this was to be loaded then load times would dramatically increase and affect the performance of the site. In summary this all depends on the size of stored files and the connection speed of the web server along with the clients connection speed. A common method implemented a while back to video streaming called buffering allows videos to be streamed on segments if your bandwidth isn't capable of keeping up with the video, the webserver continues to supply the remaining segments until you receive the entire video, a good example of a website which does this would be YouTube.

Number Of Hits

This is the number of views of certain aspects on a webserver such as most viewed pages and what pages have the most view time. These types of results are recorded and analysed so the hosts know what to add to keep clients viewing the site. This can be similar content to what has already received a lot of attention on the website. This is more of a traffic booster and not a performance booster.
